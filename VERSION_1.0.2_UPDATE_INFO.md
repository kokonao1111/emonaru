// 修正前：トークンが同じ場合は保存をスキップ
if previousToken == token {
    return  // ← ここで終了していた

// 修正後：トークンが同じでも保存を試みる
if previousToken == token {

### 2. ゲージリセット問題の修正 ⭐重要

#### 問題：
- ゲージが224/100のまま、リセットされない
- Xcode実機では正常にリセットされるが、App Store版では動作しない

#### 原因：
- リザルト画面を閉じた後、ローカルでは一時的にリセット表示
- しかし、Firestoreに保存していなかった
- 1秒後の再読み込みで元の値に戻ってしまう

#### 解決：
✅ **`FirestoreService.swift` に新メソッドを追加**
```swift
func resetPrefectureGauge(prefecture: Prefecture, currentValue: Int, 
                         maxValue: Int, completedDate: Date?) async throws {
    // Firestoreにリセット後の値を保存
}
```

✅ **`PrefectureGameMapView.swift` のリザルト画面 `onDismiss` を修正**
```swift
// リザルト画面を閉じた後
do {
    try await firestoreService.resetPrefectureGauge(
        prefecture: prefecture,
        currentValue: 0,
        maxValue: info.nextMax,
        completedDate: nil
    )
    print("✅ Firestoreにゲージリセットを保存: 0/\(info.nextMax)")
} catch {
    print("❌ ゲージリセットの保存に失敗: \(error.localizedDescription)")
}
```

✅ **結果**
- ゲージ達成後、正常に0にリセット
- アプリ再起動後もリセット状態が維持

### 3. プロフィールアイコンが変更できない問題の修正

#### 問題：
- アイコン変更時にエラー表示
- 「写真の読み込みに失敗しました」

#### 原因：
- `Info.plist` に写真ライブラリへのアクセス許可が未設定

#### 解決：
✅ **`Info.plist` に許可を追加**
```xml
<key>NSPhotoLibraryUsageDescription</key>
<string>プロフィール画像を設定するために写真ライブラリへのアクセスが必要です</string>

<key>NSPhotoLibraryAddUsageDescription</key>
<string>写真を保存するために写真ライブラリへのアクセスが必要です</string>
```

✅ **結果**
- プロフィールアイコンが正常に変更可能
- 何回でも変更可能
- 他のユーザーにも表示される
